# Status update format

The default project update. 280 characters or less, excluding formatting. The limit is a feature — it forces you to prioritize the few things stakeholders actually need.

## What goes in

A status update answers three questions, fast:

1. **Where are we?** — a clear status (on track, at risk, off track).
2. **What moved this week?** — the one or two things that matter most.
3. **What's next or blocked?** — where focus is going, or what's in the way.

If a learning, risk or decision was logged this week, it auto-surfaces under the update — so nod to it, don't re-explain it ("more in the learning below").

## Status or date changes

When the status flips or the target date moves, add one line of commentary on why. Normalizing these keeps stakeholders calm and the timeline honest.

## Overflow

If a week genuinely doesn't fit in 280, keep the 280 as the summary and add a detailed Note (the `+` → **Note** option) for the curious. It's uncapped, collapses by default, and you get one per update. Don't pad the 280 to use it up.

## Length discipline

- Count the visible characters and report the count with the draft.
- "Excluding formatting" means markdown for bold, links and emoji shortcodes doesn't count toward the limit — but verify the exact count in the composer, since what counts can be fiddly.
- Over 280 → cut, don't compress into unreadable shorthand.

## Don't

- Restate an embedded learning, risk or decision.
- Bury the status in the middle — lead with where things stand.
- Hedge. "Some progress was made on a few items" says nothing.

## Worked example

> Before wiring up state machines, I'm pausing to explore some of Claude's lesser-known first-party features — the ones most teams don't discover until much later. Sharing what I find here so we can all build on the platform more intentionally from the start.
>
> *(status: on track · target: 17 Apr · learning attached: Claude Code — Bash Mode)*

Why it works: states the pivot plainly, gives the *why* in one breath, and tees up the attached learning instead of summarizing it. Forward-looking and within the limit.
